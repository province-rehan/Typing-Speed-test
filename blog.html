The difference between sessionStorage and localStorage is that, localStorage data does not expire.On the other hand, sessionStorage data is cleared when the page session ends. Local storage is a Clint side storage data.When we store data in server side we use session storage. Page sessions are valid for only one tab at a time. So, if a user has four tabs open, each one will have its own page session. Local storage is not session-based, the data can be cleared via javascript.

A variable can be declared inside or outside a block. If a variable is declared outside all functions or curly braces, it exists in the global scope. The global variables can be accessed from anywhere in the program, including inside blocks. Whereas statement within a curly brace is a block scope and the block scope does not apply to var keyword. Const and Let can define scope to block statements.

The event loop is a mechanism that allows JavaScript to handle ascynchronous operations. When an asynchronous task is started, the event loop will start running. Once the task is completed, the event loop will again check for any other tasks that need to be executed. The Event Loop monitors the Call Stack and the Callback Queue. If the Call Stack is empty, the Event Loop will take the first event from the queue and will push it to the Call Stack ant it runs the function. Thus the event works.
